# Gas Plasma

[![CI](https://github.com/JohnThre/GasPlasma-theme/actions/workflows/ci.yml/badge.svg)](https://github.com/JohnThre/GasPlasma-theme/actions/workflows/ci.yml)
[![Pages](https://github.com/JohnThre/GasPlasma-theme/actions/workflows/pages.yml/badge.svg)](https://github.com/JohnThre/GasPlasma-theme/actions/workflows/pages.yml)
[![Open VSX](https://img.shields.io/open-vsx/v/johnthre/gas-plasma-theme?label=Open%20VSX)](https://open-vsx.org/extension/johnthre/gas-plasma-theme)

An orange-dominant dark theme for **Apple Terminal**, **iTerm2**, **VS Code**, and Open VSX-compatible editors.

Inspired by the glow of ionized gas, Gas Plasma is warm, vibrant, and easy on the eyes for long coding sessions. Colors are orange-shifted but still distinguishable for syntax highlighting, git diffs, and error output.

![Gas Plasma Screenshot](assets/mockup.svg)

## Installation

### Apple Terminal

1. Download [`GasPlasma.terminal`](GasPlasma.terminal) from this repository or the latest GitHub Release.
2. Double-click the file to open it in Terminal and add the profile automatically.
3. Go to **Terminal > Settings > Profiles** and set "Gas Plasma" as the default profile if you want it as your default.

Manual import: **Terminal > Settings > Profiles > ... > Import...** and select `GasPlasma.terminal`.

### iTerm2

1. Download [`GasPlasma.itermcolors`](GasPlasma.itermcolors) from this repository or the latest GitHub Release.
2. Open **iTerm2 > Settings > Profiles > Colors**.
3. Choose **Color Presets... > Import...** and select `GasPlasma.itermcolors`.
4. Select **GasPlasma** from **Color Presets...**.

### VS Code and Open VSX

Install the extension from Open VSX once the first release has been published:

```bash
code --install-extension johnthre.gas-plasma-theme
```

You can also install the generated VSIX from GitHub Releases:

```bash
code --install-extension johnthre.gas-plasma-theme-0.1.0.vsix
```

The extension id is `johnthre.gas-plasma-theme`.

## Features

- Shared palette source for Apple Terminal, iTerm2, VS Code, and Open VSX.
- Apple Terminal profile with macOS glass background blur, SF Mono 13pt, 120x30 window size, and bright-bold ANSI behavior.
- iTerm2 color preset using the same ANSI and UI palette.
- VS Code dark color theme with workbench, terminal ANSI, syntax token, and semantic token colors.
- Distinctive project logo and design mockup assets.

## Regenerating

Install dependencies and rebuild all generated assets:

```bash
npm ci
npm run build
npm run check
```

Create a local VSIX package:

```bash
npm run package
```

The Apple Terminal profile is generated by `generate-theme.swift` because Terminal stores colors as archived AppKit color data. The iTerm2 preset, VS Code theme, logo, screenshot, and GitHub Pages site are generated by `scripts/build.mjs` from `src/palette.json`.

## Release Signing and Publishing

Releases are designed to be created from signed `v*` tags. The release workflow verifies the tag signature, builds release artifacts, signs `SHA256SUMS` with GPG key `FEA322F2C85C0E17`, creates a GitHub Release, and publishes the VSIX to Open VSX.

Required repository secrets:

- `GPG_PRIVATE_KEY`
- `GPG_PASSPHRASE`
- `OVSX_PAT`

The Open VSX namespace is `johnthre`. If it does not exist when the release workflow runs, the workflow attempts to create it with `ovsx create-namespace`. For verified Open VSX status, claim namespace ownership after logging in to Open VSX:

https://github.com/eclipse-openvsx/openvsx/wiki/Namespace-Access

## Requirements

- macOS 26.4 or later for Apple Terminal generation.
- Node.js 22 or later for extension packaging and generated assets.
- Swift 6 or later with Xcode command-line tools for `GasPlasma.terminal`.

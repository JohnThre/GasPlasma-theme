# Security Policy

## Supported Versions

Only the latest published release is supported for security fixes.

## Reporting a Vulnerability

Report security issues privately through GitHub Security Advisories when available.

If advisories are unavailable, open a minimal issue asking for a private contact path. Do not include exploit details, tokens, credentials, private keys, or reproduction steps in a public issue.

## Scope

Security-sensitive areas include release automation, package publishing, GPG signing, dependency updates, generated extension artifacts, and any script that writes release files.

Theme color choices, visual design issues, icon coverage gaps, and documentation defects should be reported as regular issues.
